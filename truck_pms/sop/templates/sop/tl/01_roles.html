1.4 Mga Prinsipyo ng Seguridad

  • Least Privilege: Bawat role ay may pinakamababang access na kailangan para gawin ang kanilang trabaho. Walang role ang makaka-access ng feature na lampas sa kanilang antas.
  • Data Isolation: Ang mga Mechanic at Contractor ay nakakakita lamang ng Job Orders na nakatalaga sa kanila. Hindi nila kayang tingnan ang lahat ng JO sa sistema.
  • Immutable Audit Trail: Kapag na-close na ang isang Job Order, ang lahat ng data ay nakatakda sa Service Ledger at hindi na maaaring baguhin o burahin kailanman.
  • Session Timeout: Awtomatikong magla-logout ang mga user pagkatapos ng 30 minutong walang ginagawa.
  • Patakaran sa Password: Lahat ng user account ay nangangailangan ng password na hindi bababa sa 8 karakter ang haba.

Kabanata 2: Siklo ng Buhay ng User Account

2.1 Paggawa ng Bagong User

Responsableng Role: Super Admin lamang

  1. Mag-login bilang Super Admin.
  2. Pumunta sa Sidebar → User Management.
  3. I-click ang "Add User" na buton.
  4. Kumpletuhin ang form:
    • Username: Natatanging identifier (hal., jdelacruz). Gamitin ang unang letra ng pangalan + apelyido sa maliit na titik.
    • Password: Pansamantalang password. Siguraduhing hindi bababa sa 8 karakter.
    • Confirm Password: Ulatin ang parehong password.
    • Role: Pumili mula sa anim na role na nakasaad sa Kabanata 1.
    • First Name / Last Name: Legal na pangalan gaya ng nasa company ID.
    • Mobile Number: Aktibong numero para sa kontak.
    • Specialization: (Para sa Mechanics) Pangunahing lugar ng ekspertis — Engine, Brakes, Electrical, Tank at Valves, General.
    • Employee ID: ID number na itinalaga ng kompanya.
  5. I-click ang "Save".
  6. Ibigay sa bagong user ang kanilang username at pansamantalang password. Sabihan silang palitan ang password sa unang pag-login.

2.2 Pag-edit ng User

  1. Pumunta sa Sidebar → User Management.
  2. Hanapin ang user sa listahan (gamitin ang role filter kung kinakailangan).
  3. I-click ang user row o Edit button.
  4. I-update ang mga field na nais baguhin.
  5. I-click ang "Save".

2.3 Pag-reset ng Password ng User

  1. Pumunta sa Sidebar → User Management.
  2. Hanapin ang user at i-click ang key icon sa Actions column.
  3. Mare-redirect ka sa Django Admin password change form.
  4. Ilagay ang bagong password at kumpirmahin.
  5. I-click ang "Change password".
  6. Ibigay ang bagong pansamantalang password sa user nang ligtas.

2.4 Pag-deactivate ng User

  1. Pumunta sa Sidebar → User Management.
  2. I-click ang user para i-edit.
  3. Alisan ng tsek ang "Active" na checkbox.
  4. I-click ang "Save".
  5. Hindi na makakapag-login ang user, ngunit nananatili ang lahat ng kanilang historical records.

2.5 Paggawa ng OJT/Trainee Account (Espesyal)

  1. Hakbang 1: Gawin ang user account na may Role = "OJT / Trainee".
  2. Hakbang 2: Pumunta sa Sidebar → Assign Training.
  3. Piliin ang bagong gawang OJT mula sa dropdown.
  4. Magtalaga ng Supervisor (dapat Staff, Admin, o Super Admin).
  5. Itakda ang Start Date.
  6. I-click ang "Assign".
  7. Makakita na ngayon ang OJT ng kanilang Training Dashboard.

Kabanata 3: Pag-navigate sa Sistema at UI Reference

3.1 Pangkalahatang Layout

  1. Sidebar (Kaliwa): Main navigation menu. Nakatago sa mobile screens. Nagpapakita ng menu items batay sa role ng user.
  2. Top Bar: Nagpapakita ng page title, back button (kung mayroon), pangalan ng user, at role badge. Ang back button (← arrow) ay bumabalik sa dating logical page.
  3. Content Area: Pangunahing katawan ng page. Naglalaman ng mga listahan, form, detail view, at action buttons.
  4. Footer: Company tagline at impormasyon ng copyright.

3.2 Page Loader

Isang manipis na asul na progress bar ang kumikilos sa itaas ng screen habang naglo-load ang page. Ito ay nagbibigay ng visual feedback sa panahon ng pag-navigate.

3.3 Mobile Navigation

Sa maliliit na screen, nakatago ang sidebar. I-tap ang ☰ (hamburger icon) sa kaliwang sulok sa itaas para palawakin ang mobile menu.

3.4 Karaniwang UI Patterns

Back Buttons: Ang ← arrow sa top bar ay nagbabalik sa dating page. Context-aware ito.

Search Boxes: Karamihan sa mga list page ay may search box. Mag-type para i-filter ang listahan sa real time.

Filters: Ang ilang page ay may dropdown filters para sa status, type, truck, at iba pa.

Pagination: Ang mga list page ay nagpapakita ng 50 items bawat page.

CSV Export at Print: Ang PMS Schedules page ay may mga buton para sa CSV export at pag-print.

3.5 Pag-logout

  1. I-click ang "Logout" na buton sa sidebar.
  2. Lalabas ang confirmation dialog: "Are you sure you want to log out?"
  3. I-click ang OK para kumpirmahin.
  4. May lalabas na overlay na may umiikot na favicon at "Logging out..." message.
  5. Mare-redirect ka sa login page pagkatapos ng ilang segundo.
