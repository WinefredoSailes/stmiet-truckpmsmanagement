1.4 Security Principles

  • Least Privilege: Each role has the minimum access required to perform its function. No role can access features outside its permission level.
  • Data Isolation: Mechanics and Contractors can only see Job Orders assigned to them. They cannot browse all JOs in the system.
  • Immutable Audit Trail: Once a Job Order is closed, all associated data is written to the Service Ledger and can never be modified or deleted.
  • Session Timeout: Users are automatically logged out after 30 minutes of inactivity.
  • Password Policy: All user accounts require passwords of at least 8 characters.

Chapter 2: User Account Lifecycle

2.1 Creating a New User

Responsible Role: Super Admin only

  1. Log in as Super Admin.
  2. Navigate to Sidebar → User Management.
  3. Click the "Add User" button.
  4. Complete the form with the following fields:
    • Username: Unique identifier (e.g., jdelacruz). Use first initial + last name in lowercase.
    • Password: Temporary password. Ensure at least 8 characters.
    • Confirm Password: Re-enter the same password.
    • Role: Select from the six roles defined in Chapter 1.
    • First Name / Last Name: Legal name as appears on company ID.
    • Mobile Number: Active cellular number for contact purposes.
    • Specialization: (Mechanics only) Primary area of expertise — e.g., Engine, Brakes, Electrical, Tank & Valves, General.
    • Employee ID: Company-assigned employee or contractor ID number.
  5. Click "Save".
  6. Provide the new user their username and temporary password. Instruct them to change their password on first login.

2.2 Editing an Existing User

  1. Navigate to Sidebar → User Management.
  2. Locate the user in the list (use the role filter if needed).
  3. Click the user row or Edit button.
  4. Update the desired fields.
  5. Click "Save".

2.3 Resetting a User's Password

  1. Navigate to Sidebar → User Management.
  2. Find the user and click the key icon in the Actions column.
  3. You will be redirected to the Django Admin password change form.
  4. Enter the new password and confirm.
  5. Click "Change password".
  6. Communicate the new temporary password to the user securely.
⚠ Important: Password resets redirect to the Django Admin interface, which is only accessible to Super Admin accounts. If you cannot access the Django Admin, check that you are logged in as Super Admin at /admin/.

2.4 Disabling / Deactivating a User

To prevent a user from logging in without deleting their account (preserving audit trail integrity):

  1. Navigate to Sidebar → User Management.
  2. Click the user to edit.
  3. Uncheck the "Active" checkbox.
  4. Click "Save".
  5. The user can no longer log in, but all their historical records (Job Orders, service logs, ratings) remain intact.

2.5 Creating an OJT/Trainee Account (Special Case)

Trainee accounts require an additional step beyond user creation:

  1. Step 1: Create the user account with Role = "OJT / Trainee" (follow Section 2.1).
  2. Step 2: Navigate to Sidebar → Assign Training.
  3. Select the newly created OJT from the dropdown.
  4. Assign a Supervisor (must be Staff, Admin, or Super Admin).
  5. Set the Start Date (usually the OJT's first day).
  6. Click "Assign".
  7. The OJT now has access to their Training Dashboard and the supervisor can begin ratings.

Chapter 3: System Navigation & UI Reference

3.1 Layout Overview

The Truck PMS system uses a consistent layout across all pages:

  1. Sidebar (Left): Main navigation menu. Collapses on mobile screens. Shows menu items based on the user's role.
  2. Top Bar: Displays current page title, a back button (where applicable), user name, and role badge. The back button (← arrow) returns to the previous logical page.
  3. Content Area: Main body of the page. Contains lists, forms, detail views, and action buttons.
  4. Footer: Company tagline and copyright information.

3.2 Page Loader Indicator

A thin blue progress bar animates across the top of the screen whenever a page is loading. This provides visual feedback during navigation. The bar fills from left to right and fades out once the page is ready.

3.3 Mobile Navigation

On small screens (phones, small tablets), the sidebar collapses. Tap the ☰ (hamburger icon) in the top-left corner to expand the mobile menu.

3.4 Common UI Patterns

Back Buttons: The ← arrow in the top bar returns you to the previous page. This is context-aware — on a truck detail page, it returns to the truck list; on a Job Order detail, it returns to the JO list.

Search Boxes: Most list pages include a search box. Type any text to filter the list in real time. Searches match partial text against key fields (name, plate number, description, etc.).

Filters: Some pages (Job Orders, PMS Schedules) include dropdown filters to narrow results by status, type, truck, or other criteria. The filter and search work together.

Pagination: List pages display 50 items per page. Use the page controls at the bottom to navigate between pages.

CSV Export: The PMS Schedules list page includes a "CSV Export" button for downloading schedule data as a spreadsheet file.

Print: The PMS Schedules page includes a "Print" button that formats the table for browser printing (landscape orientation).

3.5 Logout Procedure

  1. Click the "Logout" button in the sidebar (Account section).
  2. A confirmation dialog appears: "Are you sure you want to log out?"
  3. Click OK to confirm.
  4. A full-screen overlay appears with a spinning favicon and "Logging out..." message.
  5. After a brief delay, you are redirected to the login page.
